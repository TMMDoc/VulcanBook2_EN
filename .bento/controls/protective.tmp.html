Parametrization of the protective glas monitoring

The protective glass monitoring is a system that monitors the condition of the protective glass. I could prevent you to produce parts of low quality or even damage the machine. T

The behavior is defined as follows:

  • When Quality value of the protective glass< error level during cutting: Machine processing will stop and beam off and then pop out a window at the front of the HMI.

  • If clickthe“Close” button of the window, then the machine will continue cutting the current whole sheet, then before process next sheet it will do a protective glass measurement then if the quality is lower than error level, it will pop out this window again. Click “Close” and process again then at the end of the sheet it will pop out this window again.

  • If theglass is contaminated at the beginning then after cutting the first contour, processing will stop and do a protective glass measurement then if the quality is lower than error level, it will pop out a window. If click the “Close” button of the window, then the machine will continue cutting the current whole sheet

There is a offline measurement and a online measurement possible:

offline: the Offline measurement will be done in the following cases

  • Start of the first program after the machine power ON equivalent to "Laser Standby" button pressed.

  • During online measurement, if found glass contaminated or contamination possible then will do it at the start of every program until the glass is Clean

Lumi measurement

The protective window measurement is based on a lumi meeasurement. If condition is “Clean”, protective window status is good to proceed with cutting in terms of good cutting results. If condition is “Contaminated”, protective window status is poor and its advisable to change the protective window to experience good cutting results.

Depending on the result of quality value percentage, clean or replace the protective glass.

  • 80%-90% → clean

  • < 80% → replace

With the above entries, X and Y axis moves towards the positioned value.

Running the program will follow the data of frequency and duty as per the user requirement. Power parameter is pre-defined as step up of 500W, 1000W and 1500W.

Focus diameter, Focus distance and dwell time also has been modified according to the user requirements. Entered values can be monitored when the program runs. Dwell time specified will hold the program until the dwell time configured and then abort.

Measure mode has offline and reference data. In offline selection, user can run the program and check the status of protective window. In reference selection, user can cross verify the data and values of quality and linearity and it can be saved using save option. Generally, reference selection will be opted when the protective window is newer which gives fine set of results.

When lumi measure program runs, it takes the log of the data and it can be read in LumiMeasurelog.txt from the path 'C:\ProgramData\Metamation\Vulcan\Log\Currentyear\Currentmonth\Currentday' or in the UI.

With reference to the above calculation, the following parameters will be added in the MK file to optimize the monitoring function:

  1. Offline broken factor

  2. Online broken factor

  3. Warning limit in percentage

  4. Error limit in percentage

  5. Delay time in secs to report warning
