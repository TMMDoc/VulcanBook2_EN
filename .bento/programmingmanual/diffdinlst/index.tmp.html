1. Introduction to LST programming format

  2. Identification of Main Program in LST From the LST file the main program is identified with the set of text that starts with ‘HP’ and ends with M30.

  1. Identification of Sub programs in LST

In the LST file, the sub programs are called from the main program with the syntax “SP@”, where @ is the number of sub program, eg: SP1 will call the respective sub program!

  1. Abstract of DIN conversion From the LST file, only the main & sub-programs are converted to DIN format with the cycle programs defined very similar to LST.

  2. Main program: The main program is defined in the DIN with N-blocks in ascending order and should not be repeated in the same program, also the end of the main program is terminated via G99.

  1. Sub-programs:

The sub programs are called from the main program using the syntax “G22 J@”, where @ is the number of the N-block defined for the start of Sub-program within the same program.

  1. Cycle programs: The cycle programs of LST are defined very similar in DIN with limitation to parameter transfer.

  1. TC_UPDATE_PARAM: A new cycle program called “TC_UPDATE_PARAM” was added for DIN format to extract the information such as LTT, Laser method, Pierce type, Cutting type of the contour from the TC_LASER_ON well in advance, so that the parameters are updated in the traverse motion. Additionally the Fly-cut, Micro joints, Nano joints, etc., information are also indicated in 4th parameter as bit wise: Bit0 – Fly cut active Bit1 – Micro-Joint active Bit2 – Nano-Joint active 5th parameter – Defines the MJ length. 6th parameter – Defines the NJ stability.

LST format

DIN format

TC_LASER_ON(1,"ST010MD0-N2SF-30-2",10,100)

G253 F="TT2: ST010MD0-N2SF-30-2" E0
TC_UPDATE_PARAM R[1;10;100;0;0;0]

  1. Start of Contour – M1014: The start of each contour was indicated as “M1014” to activate the fast outputs depends on the traverse distance, will be used for Frog Jump function, update of parameters, etc.,

  2. End of Contour – M1015: The start of each contour should be ended using “M1015” to deactivate the fast outputs depends on the last contour distance, will be used for Soft micro joints, change of cut parameter at end, etc.,

  3. Process flow of DIN conversion:

  4. In Main program:

    1. Gather all the traverse commands followed by sub program call before TC_LASER_OFF.

    2. Before the start of traverse command, add G253 parameter extracted from TC_LASER_ON of the SP1 sub program call.

    3. Now check for the TC_WRITE call before TC_LASER_ON at the beginning of the SP1 sub program. if present, then jump to the sub program call (point 2)

    4. If no TC_WRITE, then add TC_UPDATE_PARAM with the parameters extracted from TC_LASER_ON of the sub program call.

    5. After the end of traverse command or before the sub program call, add the “M1014” to indicate the start of contour.

  5. In Sub program:

    1. As flow start from the main program, the M1014 in main program should be ended by M1015 in the sub program before the TC_LASER_OFF.

    2. For next TC_LASER_ON command, add G253 & TC_UPDATE_PARAM before the traverse command, if the LTT is same then ignore the G253 command.

    3. After the end of traverse command or before the TC_LASER_ON, add the “M1014” to indicate the start of contour.

    4. Please note for single contour, there may be multiple TC_LASER_ON and the above 2 points (b & c) are only valid if M1014 is not added for that contour.

    5. For the added M1014, the M1015 command should be added before the TC_LASER_OFF.

    6. Jump to Point b.

  6. Contour Rotation:

LST format

DIN format N80 G90 N110 G01 X877.551 Y10

N120 ROT RPL=90 N130 SP1

Transfer of TT followed by H100. If there is no change in TT, then ignore this 2 lines!

G253 F=” Teil2_Mittel_Schwer” E0

TC_WRITE R[5;-90;0]

TC_WRITE is called with following parameters:
5 – text height in mm
-90 – text height in degrees
0 – text box length (default 0)

TC_LASER_OFF R[1]

G253 F="TT2: ST030MD0-O2S0-30-2" E0

In above, the G253 command is used for the text to be written, so for the next TC_UPDATE_PARAM then again add the TT transfer, else there will be error!

  1. Soft Micro-Joints & Nano-Joints: The Micro-joints & Nano-Joints information are defined in the LST under the MICROJOINT table as shown below.

  1. Activation of MJ & NJ in TC_UPDATE_PARAM – 4th parameter as follows:

    1. Bit1 – Micro-Joint active.

    2. Bit2 – Nano-Joint active.

  2. Micro joints – Combination of TC_MICROJOINT(“MIJO-1”) & G821 command in contour, the length of micro joint depends on Contour type is defined in the TC_UPDATE_PARAM – 5th parameter.

  3. Nano joints stability is defined in the TC_UPDATE_PARAM – 6th parameter.
